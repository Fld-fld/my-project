## 第3页：项目背景

### 问题提出

随着电子商务的快速发展，校园快递业务量急剧增长，传统的快递取件模式已无法满足师生需求：

- 🚶 **取件拥堵**：高峰时段站点人流密集，学生排队时间长
- 📱 **信息滞后**：学生无法及时获知快递到达情况
- 📝 **管理低效**：手工录入和核对工作量大、易出错
- ⏰ **缺乏预约**：无取件预约机制，导致拥堵加剧

---

## 第4页：项目意义

### 研究意义

| 目标 | 说明 |
|------|------|
| 🔄 **分流减压** | 通过预约功能引导错峰取件，缓解站点压力 |
| 👁️ **信息透明** | 实时查询快递状态，减少无效等待 |
| ⚡ **效率提升** | 系统化管理快递信息，降低人工成本 |
| 😊 **体验优化** | 消息通知、预约提醒等功能提升服务质量 |

---

## 第5页：可行性分析

### 三大可行性

1. **技术可行性**
   - Spring Boot + Vue.js 成熟技术栈
   - 前后端分离架构，开发效率高
   - MySQL 稳定可靠

2. **经济可行性**
   - 开源技术，无版权费用
   - 部署成本低，维护方便

3. **社会可行性**
   - 解决校园快递痛点，具有实际应用价值
   - 提升校园服务质量

## 第12页：数据库设计（核心表）

### 快递表 (express)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键ID |
| tracking_number | VARCHAR(50) | 快递单号 |
| company | VARCHAR(30) | 快递公司 |
| recipient_name | VARCHAR(30) | 收件人姓名 |
| recipient_phone | VARCHAR(20) | 收件人手机号 |
| location | VARCHAR(50) | 存放位置 |
| pickup_code | VARCHAR(20) | 取件码 |
| status | INT | 状态（0待取/1已取） |
| admin_id | BIGINT | 录入管理员 |

## 第13页：系统首页

首页采用移动端优先的设计风格，整体布局简洁清爽，使用蓝色作为主色调。

- 🎠 **轮播图**：展示最新公告和取件提醒，支持自动播放和手动切换
- 🔍 **搜索框**：支持按取件码或快递单号查询
- 📌 **功能入口**：我的快递、取件预约、消息通知三个卡片
- 📦 **待取快递快捷区**：直接展示最近2个待取快递的取件码和存放位置

## 第14页：注册登录

### 登录页面

登录页面采用居中卡片式布局，包含用户名和密码输入框，支持点击忘记密码跳转到重置页面。

![登录](images/图5-2登录.png)

### 注册页面

注册页面需要输入姓名、学号、手机号和密码，表单带有实时验证功能。系统采用 JWT 令牌进行身份认证，登录成功后将令牌存储在本地存储中。

![注册](images/图5-3注册.png)

## 第15页：我的快递

用户可查看所有快递记录，每张卡片突出显示取件码，支持一键复制。快递列表按时间倒序排列，每张卡片显示快递公司、快递单号、取件码、存放位置和时间等信息。

状态标签采用彩色区分：
- 🟠 **橙色**：待取件
- 🔵 **蓝色**：已预约
- 🟢 **绿色**：已取件

卡片支持点击操作和状态查看详情。

![我的快递](images/图5-4我的快递.png)

---

## 第16页：取件预约

预约页面显示所有待取快递，用户选择快递后可预约取件时间段，系统显示各时段预约人数，帮助用户选择合适的时间。

预约流程：
1. 选择待取快递
2. 选择预约日期
3. 选择时间段
4. 提交预约

![取件预约](images/图5-5取件预约.png)

## 第17页：消息通知

系统向用户推送快递到件、预约提醒等通知消息，支持一键标记已读。通知列表按时间倒序排列，未读通知显示红色圆点标记。

功能特点：
- 🔴 **未读标记**：未读通知显示红色圆点
- ✅ **批量已读**：一键标记全部已读
- 🔔 **实时推送**：快递状态变更时自动推送通知

## 第20页：管理端后台首页

管理员登录后进入后台首页，显示数据统计卡片和快捷操作。

统计数据：
- 📦 **总快递数**：系统所有快递数量
- ⏳ **待取件数**：等待领取的快递数量
- ✅ **已取件数**：已领取的快递数量
- 📈 **取件率**：已取件/总快递数百分比
- 📥 **今日到件**：今日新到的快递数量
- 📅 **今日预约**：今日的预约数量

快捷操作：录入快递、快递管理、预约管理、用户管理

管理员可单条录入或批量导入快递信息。

- 快递单号
- 快递公司
- 收件人姓名
- 收件人手机号
- 存放位置

### 批量录入

支持粘贴数据批量导入，格式说明：
```
快递单号 快递公司 收件人姓名 收件人手机 存放位置
```

## 第22页：快递管理

管理员可查看所有快递记录，支持按单号、状态、快递公司筛选搜索。
