# 校园快递取件优化系统 - 数据库设计文档

## 1. 数据库概念模型（E-R 图）

### 1.1 实体关系图

```
┌─────────────┐         ┌─────────────┐
│   用户表    │ 1    n  │  预约表     │
│   (user)    │─────────│(reservation)│
└─────────────┘         └─────────────┘
       │                        │
       │ 1                      │ n
       │                        │ 1
       │                        │
       │         ┌──────────────┘
       │         │
       ▼         ▼
┌─────────────┐         ┌─────────────┐
│  通知表     │ n    1  │   快递表    │
│(notification)│─────────│  (express)  │
└─────────────┘         └─────────────┘
                              │
                              │ n
                              │
                              │ 1
                              ▼
                       ┌─────────────┐
                       │  管理员表   │
                       │  (admin)    │
                       └─────────────┘
```

### 1.2 实体说明

| 实体 | 描述 | 主要属性 |
| :--- | :--- | :--- |
| **用户 (User)** | 系统使用者（学生） | 学号、姓名、手机号、密码 |
| **管理员 (Admin)** | 后台管理者 | 用户名、密码、权限级别 |
| **快递 (Express)** | 快递包裹信息 | 快递单号、快递公司、收件人、取件码、状态 |
| **预约 (Reservation)** | 取件预约记录 | 用户、快递、预约时间、状态 |
| **通知 (Notification)** | 系统通知消息 | 用户、通知类型、内容、已读状态 |

### 1.3 实体关系说明

| 关系 | 类型 | 说明 |
| :--- | :--- | :--- |
| 用户 - 预约 | 1:n | 一个用户可以有多个预约记录 |
| 快递 - 预约 | 1:n | 一个快递可以有多个预约记录（历史） |
| 快递 - 管理员 | n:1 | 多个快递可以由一个管理员录入 |
| 用户 - 通知 | 1:n | 一个用户可以接收多条通知 |

## 2. 数据库逻辑模型

### 2.1 用户实体

```
用户 (User)
├── id (主键)
├── studentId (学号，唯一)
├── name (姓名)
├── phone (手机号，唯一)
├── password (密码，MD5加密)
├── role (角色)
├── createdAt (创建时间)
└── updatedAt (更新时间)
```

### 2.2 管理员实体

```
管理员 (Admin)
├── id (主键)
├── username (用户名，唯一)
├── password (密码，MD5加密)
├── role (角色：ADMIN)
├── createdAt (创建时间)
└── updatedAt (更新时间)
```

### 2.3 快递实体

```
快递 (Express)
├── id (主键)
├── trackingNumber (快递单号，唯一)
├── company (快递公司)
├── recipientName (收件人姓名)
├── recipientPhone (收件人手机号)
├── location (存放位置)
├── pickupCode (取件码，唯一)
├── status (状态)
├── adminId (录入管理员 ID)
├── createdAt (创建时间)
└── updatedAt (更新时间)
```

### 2.4 预约实体

```
预约 (Reservation)
├── id (主键)
├── userId (用户 ID，外键)
├── expressId (快递 ID，外键)
├── reservationDate (预约日期)
├── startTime (开始时间)
├── endTime (结束时间)
├── status (状态)
├── createdAt (创建时间)
└── updatedAt (更新时间)
```

### 2.5 通知实体

```
通知 (Notification)
├── id (主键)
├── userId (用户 ID，外键)
├── type (通知类型)
├── title (标题)
├── content (内容)
├── isRead (是否已读)
├── createdAt (创建时间)
└── updatedAt (更新时间)
```

## 4. 索引设计

### 4.1 已建索引（主键和唯一约束自动创建）

| 表名 | 索引名 | 字段 | 类型 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| user | PRIMARY | id | 主键索引 | 主键自动创建 |
| user | uk_student_id | student_id | 唯一索引 | 学号唯一 |
| user | uk_phone | phone | 唯一索引 | 手机号唯一 |
| admin | PRIMARY | id | 主键索引 | 主键自动创建 |
| admin | uk_username | username | 唯一索引 | 用户名唯一 |
| express | PRIMARY | id | 主键索引 | 主键自动创建 |
| express | uk_tracking_number | tracking_number | 唯一索引 | 快递单号唯一 |
| express | uk_pickup_code | pickup_code | 唯一索引 | 取件码唯一 |
| reservation | PRIMARY | id | 主键索引 | 主键自动创建 |
| notification | PRIMARY | id | 主键索引 | 主键自动创建 |

### 4.2 需要创建的索引

```sql
-- 快递表：按收件人手机号查询（用户查询自己的快递）
CREATE INDEX idx_recipient_phone ON express(recipient_phone);

-- 预约表：按用户 ID 查询（用户查询自己的预约）
CREATE INDEX idx_user_id ON reservation(user_id);

-- 预约表：按快递 ID 查询（查询快递的预约记录）
CREATE INDEX idx_express_id ON reservation(express_id);

-- 通知表：按用户 ID 查询（用户查询自己的通知）
CREATE INDEX idx_user_id ON notification(user_id);
```

### 4.3 索引设计理由

| 索引 | 理由 |
| :--- | :--- |
| `idx_recipient_phone` | 用户登录时需要查询自己名下的所有快递，使用频率高 |
| `idx_user_id` (预约表) | 用户查询自己的预约记录 |
| `idx_express_id` (预约表) | 查询快递的预约记录 |
| `idx_user_id` (通知表) | 用户查询自己的通知列表 |

## 5. 数据库初始化脚本

### 5.1 创建数据库

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS campus_express 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE campus_express;
```

### 5.2 创建数据表

```sql
-- 用户表
CREATE TABLE `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `student_id` VARCHAR(20) NOT NULL,
  `name` VARCHAR(50) NOT NULL,
  `phone` VARCHAR(11) NOT NULL,
  `password` VARCHAR(255) NOT NULL,
  `role` VARCHAR(20) NOT NULL DEFAULT 'STUDENT',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_student_id` (`student_id`),
  UNIQUE KEY `uk_phone` (`phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';

-- 管理员表
CREATE TABLE `admin` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password` VARCHAR(255) NOT NULL,
  `role` VARCHAR(20) NOT NULL DEFAULT 'ADMIN',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='管理员表';

-- 快递表
CREATE TABLE `express` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `tracking_number` VARCHAR(50) NOT NULL,
  `company` VARCHAR(50) NOT NULL,
  `recipient_name` VARCHAR(50) NOT NULL,
  `recipient_phone` VARCHAR(11) NOT NULL,
  `location` VARCHAR(50) NOT NULL,
  `pickup_code` VARCHAR(10) NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  `admin_id` BIGINT DEFAULT NULL,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_tracking_number` (`tracking_number`),
  UNIQUE KEY `uk_pickup_code` (`pickup_code`),
  KEY `idx_recipient_phone` (`recipient_phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='快递表';

-- 预约表
CREATE TABLE `reservation` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `express_id` BIGINT NOT NULL,
  `reservation_date` DATE NOT NULL,
  `start_time` TIME NOT NULL,
  `end_time` TIME NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_express_id` (`express_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='预约表';

-- 通知表
CREATE TABLE `notification` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `title` VARCHAR(100) NOT NULL,
  `content` TEXT NOT NULL,
  `is_read` TINYINT(1) NOT NULL DEFAULT 0,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='通知表';
```

## 6. 数据库设计说明

### 6.1 设计原则

1. **第三范式（3NF）**：所有表都满足第三范式，消除数据冗余
2. **字符集统一**：使用 utf8mb4，支持 emoji 等特殊字符
3. **时间字段**：使用 DATETIME 类型，自动记录创建和更新时间
4. **状态字段**：使用 VARCHAR 而非数字，提高可读性
5. **密码加密**：使用 MD5 加密存储密码

### 6.2 实体类与数据库字段映射

项目使用 MyBatis 的 `map-underscore-to-camel-case: true` 配置，自动将数据库的下划线命名映射到 Java 的驼峰命名：

| 数据库字段 | Java 属性 | 类型 |
| :--- | :--- | :--- |
| student_id | studentId | String |
| created_at | createdAt | LocalDateTime |
| updated_at | updatedAt | LocalDateTime |
| tracking_number | trackingNumber | String |
| recipient_name | recipientName | String |
| recipient_phone | recipientPhone | String |
| pickup_code | pickupCode | String |
| reservation_date | reservationDate | LocalDate |
| start_time | startTime | LocalTime |
| end_time | endTime | LocalTime |
| is_read | isRead | Integer |
