## 2. 系统业务分析

### 2.1 业务背景

随着高校学生数量的增加和电商的快速发展，校园快递业务量持续增长。传统的快递取件模式已无法满足师生的需求，主要问题包括：

1. **快递集中到达**：每日10:00-12:00、15:00-17:00为取件高峰期
2. **排队时间长**：高峰期排队时间可达30-60分钟
3. **信息不及时**：学生无法实时获知快递状态
4. **管理难度大**：快递点人工管理效率低，容易出错

### 5.3 用例详细说明

#### 用例：用户登录

| 属性 | 内容 |
| :--- | :--- |
| **用例名称** | 用户登录 |
| **参与者** | 学生用户 |
| **前置条件** | 用户已注册账号 |
| **后置条件** | 用户登录成功，系统生成JWT令牌 |
| **基本流程** | 1. 用户进入登录页面<br>2. 输入手机号和密码<br>3. 点击登录按钮<br>4. 系统验证账号密码<br>5. 验证成功，跳转到首页 |
| **异常流程** | 1. 账号或密码错误：提示错误信息<br>2. 账号被锁定：提示锁定时间 |

#### 用例：预约取件

| 属性 | 内容 |
| :--- | :--- |
| **用例名称** | 预约取件 |
| **参与者** | 学生用户 |
| **前置条件** | 用户已登录，有未取件的快递 |
| **后置条件** | 预约成功，系统发送确认通知 |
| **基本流程** | 1. 用户进入"我的快递"页面<br>2. 选择要预约的快递<br>3. 点击"预约取件"按钮<br>4. 选择取件时间段<br>5. 确认预约信息<br>6. 系统保存预约记录 |
| **异常流程** | 1. 时间段已满：提示选择其他时间<br>2. 快递已被取走：提示无法预约 |

#### 用例：快递录入

| 属性 | 内容 |
| :--- | :--- |
| **用例名称** | 快递录入 |
| **参与者** | 管理员 |
| **前置条件** | 管理员已登录后台 |
| **后置条件** | 快递信息保存成功，生成取件码 |
| **基本流程** | 1. 管理员进入快递录入页面<br>2. 输入快递信息（单号、快递公司、收件人等）<br>3. 选择存放位置<br>4. 点击保存按钮<br>5. 系统生成取件码并保存记录<br>6. 系统发送通知给收件人 |
| **异常流程** | 1. 快递单号重复：提示已存在<br>2. 收件人不存在：提示无法匹配 |
