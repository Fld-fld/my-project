## 2. 核心业务逻辑

### 2.1 取件码生成算法

```java
// 取件码 = 时间戳后4位 + 随机2位
public static String generateCode() {
    String timestamp = String.valueOf(System.currentTimeMillis());
    String timePart = timestamp.substring(timestamp.length() - 4);
    String randomPart = String.format("%02d", random.nextInt(100));
    return timePart + randomPart;
}
```

### 2.2 预约时间段冲突检测

```java
// 规则：
// 1. 每时段最多20人
// 2. 用户每天只能有1个有效预约
// 3. 预约时间前1小时不可取消

if (count >= MAX_CAPACITY_PER_SLOT) {
    throw new BusinessException("该时段已约满");
}
if (userCount > 0) {
    throw new BusinessException("您当天已有预约");
}
```

### 2.3 快递状态流转

```
PENDING (待取) → RESERVED (已预约) → PICKED_UP (已取件)
```

## 5. 核心SQL语句

### 5.1 查询用户快递

```sql
SELECT * FROM express 
WHERE recipient_phone = #{recipientPhone} 
ORDER BY created_at DESC
```

### 5.2 预约时段冲突检测

```sql
SELECT COUNT(*) FROM reservation 
WHERE reservation_date = #{date} 
  AND start_time = #{startTime} 
  AND status != 'CANCELLED'
```

### 5.3 统计快递状态

```sql
SELECT COUNT(*) FROM express WHERE status = #{status}
```

### 5.4 发送通知

```sql
INSERT INTO notification (user_id, type, title, content, is_read) 
VALUES (#{userId}, #{type}, #{title}, #{content}, 0)
```

### 5.5 更新快递状态

```sql
UPDATE express SET status=#{status} WHERE id=#{id}
```
